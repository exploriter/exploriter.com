<a href="/concept/conversation-types">**Debate**</a> is a 2-way, competitive (win-lose), argumentative engagement focused on finding flaws with the
goal of persuasion and rhetorical victory.

1. **Present** arguments (_to build a case_).
2. **Attack** arguments (_to tear down the competing case_).
3. **Defend** and repair arguments (_to protect from attack_).

Because debate happens linearly, these three things quickly become difficult to follow for both the participants and the audience. The debate tends toward being asymmetric (someone always gets the first and last word), incomplete, chaotic, disorganized, disengaged, inefficient, equivocal, ambiguous, and prone to gamesmanship (ad hominem, strawman, dodging, etc.).

However, unlike dialogue, debate lends itself to a stricter structure so we can use tools and technology to move from a state of chaos (obscurity) to order (clarity).

For example[^debate], live visual aids of the debate tree structure so everyone can follow along as arguments get sorted into agreements and disagreements, which makes it challenging to obscure anything (and obvious when it occurs).

[^debate]: Analysis based on <a href="http://theqparq.com/" target="_blank">The QPARQ</a>.
